## English

### 1) What is this project?

This project is a **WhatsApp group administration bot** that works in an **unofficial** way through **WhatsApp Web** using the **Baileys** library (meaning: it does not need Chrome or a browser-based WhatsApp Web session).

* It runs on **one WhatsApp account only** (one number) at a time.
* It is designed to be used **inside groups only** (private chats will reject commands).
* The botโs replies inside WhatsApp are **in Arabic**.
* The commands themselves are short keywords like `!kick` and `!menu` (and you can change the starting prefix).

---

### 2) Very important before using it (Unofficial)

This bot is **not an official WhatsApp app** and it does not use WhatsAppโs official APIs.

* It may stop working or change behavior because of WhatsApp updates.
* The account running the bot may face **limitations, logouts, or bans**.
* The project does not include any โtricksโ to bypass WhatsApp policies or hide automation; it relies on conservative behavior like cooldowns.

Practical tip:

* If this bot will run 24/7 or enforce strict moderation, consider using a dedicated WhatsApp number for the bot instead of your main personal number.

---

### 3) Where can you run it? (Local or VPS)

This project **does NOT require a VPS** to work.

You can run it on:

* **Your own computer** (PC/laptop).
* **A VPS** (a server that stays online all the time).

Why is a VPS โrecommendedโ?

* Because a server usually stays running 24/7 without sleeping or shutting down.
* It is good for groups that need always-on moderation (anti-link / deleting messages / welcome messages).

When is your personal computer enough?

* If you want to test the bot or use it in a small group.
* If you only need the bot for certain periods (for example, during work hours).

Quick examples:

* Your computer: โI run the bot for an hour or two when I need itโ.
* VPS: โI want the bot to watch the group all the time and handle new members all dayโ.

### 12) How does the bot work internally? (Simple explanation)

This is a simplified workflow:

1. On startup:

* The bot reads ENV variables.
* It loads/creates `store.json`.
* It loads/creates `auth-state.json`.
* It connects to WhatsApp using Baileys.

2. On first run or after logout:

* It prints a QR in the terminal to login.
* After scanning, it saves the session so future runs do not need QR.

3. While running in groups:

* When a message arrives:

  * It first enforces mutes (if the sender is muted: it tries to delete the message).
  * Then it enforces moderation rules (links/filter/images/stickers) based on the group config.
  * Then it tries to parse the message as a command (like `!menu`).
* When a new participant joins:

  * If they are banned in that group: it tries to remove them automatically.
  * If welcome is enabled: it sends a welcome message (optionally with a rules summary).

4. On disconnect:

* It tries to reconnect automatically with increasing delays (best effort).
* If it detects an explicit โlogged outโ: it stops and requires relinking.

### 14) Rate limiting and anti-spam

To avoid flooding the group:

1. General command cooldown (`BOT_CMD_COOLDOWN_MS`)

* Stops the same person from using commands too quickly in the same group.
* If set to `0`, the cooldown is disabled.

2. Fun command cooldown (`BOT_FUN_COOLDOWN_MS`)

* Extra cooldown only for fun commands.
* It works on top of the general command cooldown (both can apply).

3. Warning cooldown (`BOT_MOD_WARN_COOLDOWN_MS`)

* Avoids sending many warnings quickly to the same person for the same reason (link/filter/image/sticker/mute).

What will you see during cooldown?

* A message like: `โณ ุงูุชุธุฑ X ุซุงููุฉ ูุจู ...`

### 17) Common troubleshooting

1. โPrivileged commands are rejected for everyoneโ

* Common cause: `BOT_ALLOWLIST` is empty (or not synced).
* Fix:

  * Put your number in `BOT_ALLOWLIST` and restart.
  * Or edit `store.json` (if you know how) or reset storage.

2. โCannot run the command because the bot is not an admin.โ

* Cause: the bot account is not a group admin.
* Fix: make the bot account a group admin.

3. โViolations/muted messages are not deletedโ

* Common cause: message deletion usually requires admin privileges.
* Fix: make the bot a group admin, or expect deletions to fail sometimes.

4. โInteractive menu does not showโ

* This can happen on some devices/WhatsApp versions.
* Fix: use the text menu (send `1`/`2`/`3`/`4` after `!menu`).

5. โQR shows every timeโ

* `BOT_AUTH_DIR` may not be writable or may be deleted.
* Fix:

  * Make sure the path is correct and writable.
  * Do not run the bot in a mode that cannot save files.

6. โThe program exits saying Node.js is unsupportedโ

* Fix: upgrade Node.js to version 20 or newer.

7. โIt tells me to wait (cooldown)โ

* Fix:

  * Wait as requested, or
  * reduce cooldown values in ENV (for example `BOT_CMD_COOLDOWN_MS=0` for testing).
